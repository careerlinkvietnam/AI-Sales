# AI-Sales プロジェクト

## プロジェクト概要

**AI-Sales** は CareerLink CRM と連携した営業自動化システムです。

- **目的**: タグベースで企業を検索 → メール下書き作成 → Slack通知 → CRM記録
- **言語**: TypeScript (Node.js)
- **制約**: Gmail送信禁止（draftのみ）、CRM書き込みは許可制

---

## セッション開始時の必須作業

### 1. まず読むファイル
```
docs/SESSION_HANDOFF.md
```
- 全ルール（1〜17 + サブルール）
- 企業処理ワークフロー
- メール作成ルール
- 処理済み企業リスト

### 2. 現在のタスク確認
- 「南部・X月連絡」タグの企業を処理中
- SESSION_HANDOFF.md の「処理済み企業」「未処理企業」を確認

---

## 企業処理ワークフロー（概要）

```
Step 0: CRMで直接タグ確認 ★必須★
    ↓
Step 1-4: 企業情報・連絡履歴を確認
    ↓
Step 5-6: メール本文を作成 → 件名を決める
    ↓
Step 7-8: Gmail下書き保存 + Slack通知
    ↓
Step 9: ユーザーがメール送信
    ↓
Step 10: CRM登録 → タグ更新
```

## 重要ルール（厳守）

### ルール2: 常にCRMで直接確認
- スクリプト結果やキャッシュを鵜呑みにしない
- **処理直前にCRM画面で確認**（状況は常に変化する）

### ルール14-B: お客様に刺さるメールを作成
- **テンプレートは参考例。そのまま使わない**
- 確認すべき情報:
  1. Gmail送信履歴（過去のメール内容）
  2. CRMコールメモ（電話のやり取り）
  3. CRM訪問メモ（訪問時の記録）
- 「このお客様に何を伝えれば響くか」を考える

### メール作成ルール
1. **本文を先に書く**
2. **内容に合った件名を作成**
3. **件名フォーマット**: `[内容]【キャリアリンク佐藤】`
   - 全角カッコ【】使用
   - スペースなしで直接つなげる

### ルール15-16: スキップ条件
- 「求人受領中」タグ → スキップ（Slack通知 + タグ更新）
- 「過去求人受領」+ 3ヶ月以内に連絡済み → スキップ（Slack通知 + タグ更新）

## 主要スクリプト

| スクリプト | 用途 |
|-----------|------|
| `search_sent_to.ts "@domain"` | Gmail送信履歴検索 |
| `get_company_detail.ts {ID}` | 企業詳細取得 |
| `get_company_history.ts {ID}` | 連絡履歴取得 |
| `create_email_draft.ts {json}` | 下書き作成+Slack通知 |
| `notify_skip.ts` | スキップ通知 |
| `update_month_tag.ts {ID}` | 月タグ更新（+3ヶ月） |
| `list_drafts.ts` | Gmail下書き一覧 |

---

## 禁止事項

1. **テンプレートをそのままコピーして使わない**
2. **署名を勝手に変更しない**
3. **宛名から会社名を省略しない**
4. **CRMを確認せずに作業しない**
5. **ユーザーの許可なくCRMに書き込まない**
6. **件名の【キャリアリンク佐藤】を変更しない**
